Практическая работа №5

Тема работы

Название: Отладка отдельных модулей программного проекта

Цели и задачи

Задачи:

  • приобрести навыки работы в команде по средством использования системы контроля версий;
  • получить навыки разработки клиент-серверных приложений;
  • изучить отладку отдельных модулей программного проекта.

Пояснение к практической работе

Практическая работа выполняется в команде из двух человек.
Взаимодействие между участниками происходит посредством системы контроля версий.
При отсутствии фиксаций изменений одного из участников команды, выполнение работы не засчитывается для него.

Видео выполнения практической работы

Порядок выполнения работы

Менеджер лицензий.


Необходимо реализовать сервер, который:
1) при обращении клиента генерирует уникальный код. Уникальный код является идентификатором для последующей взаимосвязи клиента и сервера;
2) при обращении клиента проверяет, есть ли место для лицензирования клиента. Пример: возможно авторизированных клиентов 5, при подключении 6 клиента, необходимо сообщить клиенту о нехватке места;
3) отображает количество клиентов, подключенных к серверу, выводит такую информацию как: дата и время подключения, продолжительность подключения, IP-адрес клиента, порт клиента, уникальный код подключения;
4) по истечению N-ого времени отключает клиента, для обновления уникального кода;
5) имеет возможность при вводе команды отключить тот или иной клиент;
6) при первоначальном запуске, запрашивает у пользователя настройки, такие IP-адрес и порт сервера, кол-во уникальных кодов при подключении, время отключение клиента, время подтверждения уникальный код.

Необходимо реализовать клиент, который:
1) выполняет запрос для получения уникального кода;
2) при первоначально запуске, запрашивает у пользователя настройки, такие как IP-адрес и порт сервера;
3) выводит такую информацию как: дата и время подключения, продолжительность подключения, IP-адрес клиента, порт клиента, уникальный код подключения;
4) периодически выполняет запросы на обновление уникального кода и на проверку не заблокирован ли он.

При разработке можно использовать: WPF, Console, ASP.NET. UDP, TCP в качестве транспортных протоколов. MSSQL и MYSQL в качестве базы данных.

Дополнительно

  • Для получения оценки «Хорошо», необходимо реализовать добавление клиента в чёрный список. При подключении клиента, проверять находится ли он в чёрном списке, возвращать клиенту ошибку.
  • Для получения оценки «Отлично», необходимо реализовать функционал на оценку «хорошо», а также реализовать сервер, который: при обращении клиента проверяет полученный логин и пароль пользователя, хранящийся в базе данных. При прохождении аутентификации, генерирует уникальный код.
